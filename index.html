Arrays

Arrays are a collection of elements stored in contiguous memory locations, allowing efficient access using an index.

They have a fixed size, and insertion or deletion can be expensive due to shifting elements.

  • Provide constant-time access to elements using an index.
    • Linked-lists

    Linked lists are a sequence of nodes where each node contains data and a reference to the next node

    They allow dynamic memory allocation but have slower access times compared to arrays.

  • Efficient insertion and deletion at any position without shifting elements
    • Stack

    A stack is a linear data structure that follows the LIFO (Last In, First Out) principle.

    Operations such as push (insert) and pop (remove) occur only at the top of the stack.

  • Ideal for managing recursive calls and undo operations using the LIFO principle.
    • Queue

    A queue is a linear data structure that operates on the FIFO (First In, First Out) principle.

    It supports enqueue (add) at the rear and dequeue (remove) at the front.

  • Best suited for scheduling tasks and managing resources using the FIFO principle.
    • Graph

    A graph is a collection of nodes (vertices) connected by edges, which may be directed or undirected.

    Graphs are used to model relationships like networks, dependencies, or paths.

  • Represents complex relationships like social networks or road maps effectively.
